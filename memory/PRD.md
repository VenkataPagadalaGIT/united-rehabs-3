# United Rehabs - Product Requirements Document

## Original Problem Statement
Migrate the United Rehabs website from the "lovable" platform (using Supabase) to a new, clean, and scalable architecture. The new project should be in a new git repository named `united-rehabs-3`. 

### Core Requirements
1. Full migration from Lovable/Supabase to a new stack (FastAPI, React, MongoDB)
2. Replicate all features of the old site, including a comprehensive admin panel
3. Implement admin-only authentication
4. Develop a multi-agent AI pipeline (Research, Generate, Fact-Check, QA) to populate state-specific addiction data
5. Implement a feature for manual bulk data import
6. Ensure data generated by the pipeline is stored in the database and reused by the frontend to minimize API calls
7. **DATA ACCURACY IS CRITICAL** - All statistics must be verified against authoritative government sources

### Verification Metadata Stored
Each verified record now includes:
- `verified_at` - Timestamp of verification
- `last_verification_date` - Last verification date
- `next_verification_due` - Next scheduled verification
- `data_source` - Primary data source name
- `data_source_url` - Link to official source
- `data_source_agency` - Issuing government/organization
- `reliability_score` - 1-10 rating
- `verification_method` - bulk_import, trend_extrapolation, serp_verification
- `cross_check_sources` - Array of sources used for validation
- `verification_history` - Audit trail of all changes

## Architecture

### Tech Stack
- **Frontend**: React 18, TypeScript, Vite, TailwindCSS, Shadcn/ui, Axios, React Query
- **Backend**: FastAPI, Python 3.11, Pydantic
- **Database**: MongoDB
- **Authentication**: JWT with bcrypt password hashing
- **AI Integration**: Google Gemini 2.5 Flash via Emergent LLM Key

### Directory Structure
