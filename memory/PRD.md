# United Rehabs - Product Requirements Document

## Original Problem Statement
Migrate the United Rehabs website from the "lovable" platform (using Supabase) to a new, clean, and scalable architecture. The new project should be in a new git repository named `united-rehabs-3`. 

### Core Requirements
1. Full migration from Lovable/Supabase to a new stack (FastAPI, React, MongoDB)
2. Replicate all features of the old site, including a comprehensive admin panel
3. Implement admin-only authentication
4. Develop a multi-agent AI pipeline (Research, Generate, Fact-Check, QA) to populate state-specific addiction data
5. Implement a feature for manual bulk data import
6. Ensure data generated by the pipeline is stored in the database and reused by the frontend to minimize API calls

## 4-Agent Pipeline Architecture

```
ORCHESTRATOR (server.py - POST /api/pipeline/run)
    │
    ├── AGENT 1: Research      → Gemini 2.5 Flash (data gathering)
    ├── AGENT 2: Content Gen   → GPT-4o (content creation)
    ├── AGENT 3: Fact Checker  → Claude Sonnet 4.5 (cross-validation)
    └── AGENT 4: QA Agent      → No LLM (database verification)
```

**Why 3 Different LLMs?**
- Better data validation through cross-model verification
- Reduces single-model bias
- Each model optimized for its task:
  - Gemini: Fast, cost-effective for research
  - GPT-4o: Strong content generation
  - Claude: Excellent for analysis/fact-checking

### How to Run Pipeline
```bash
curl -X POST "/api/pipeline/run" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"state_name": "Texas", "state_abbrev": "TX", "years": [2024]}'
```

## Architecture

### Tech Stack
- **Frontend**: React 18, TypeScript, Vite, TailwindCSS, Shadcn/ui, Axios, React Query
- **Backend**: FastAPI, Python 3.11, Pydantic
- **Database**: MongoDB
- **Authentication**: JWT with bcrypt password hashing
- **AI Integration**: Google Gemini 2.5 Flash via Emergent LLM Key

### Directory Structure
```
/app/
├── backend/
│   ├── auth.py             # JWT Authentication logic
│   ├── bulk_import.py      # Manual data import endpoint
│   ├── content_generator.py # AI Content Generation (Gemini)
│   ├── data_pipeline.py    # Multi-agent data pipeline
│   ├── models.py           # Pydantic models for DB collections
│   ├── server.py           # Main FastAPI server and all API routes
│   └── requirements.txt
├── frontend/
│   ├── src/
│   │   ├── components/     # Reusable UI components
│   │   │   ├── listing/    # Public page components (Header, Footer, Tabs, FAQ)
│   │   │   └── ui/         # Shadcn UI components
│   │   ├── hooks/          # Custom React hooks (useAuth, etc.)
│   │   ├── lib/            # API client (api.ts)
│   │   ├── pages/          # Page components
│   │   │   ├── admin/      # Admin panel pages (14 pages)
│   │   │   ├── StatePage.tsx        # Router for state pages
│   │   │   ├── StateStatsPage.tsx   # Statistics page
│   │   │   ├── StateRehabsPage.tsx  # Rehabs listing page
│   │   │   └── StateResourcesPage.tsx # Free resources page
│   │   └── data/           # State configuration
│   ├── package.json
│   └── vite.config.ts
└── memory/
    └── PRD.md
```

### Key Database Collections
- `users` - Admin users with hashed passwords
- `state_addiction_statistics` - State-level addiction statistics by year
- `substance_statistics` - Substance-specific statistics
- `free_resources` - Treatment resources (nationwide and state-specific)
- `faqs` - FAQs (general and state-specific)
- `articles` - Blog posts and articles
- `page_seo` - SEO metadata for pages
- `page_content` - Dynamic page content

## Key API Endpoints
- `GET /api/statistics?state_id=FL` - Get statistics for a state
- `GET /api/resources?state_id=FL` - Get resources (includes nationwide)
- `GET /api/faqs?state_id=FL` - Get FAQs for a state
- `GET /api/substance-statistics?state_id=FL` - Get substance-specific statistics
- `POST /api/pipeline/run` - Run AI data pipeline for a state
- `POST /api/bulk/import` - Import bulk data manually
- `POST /api/auth/login` - Admin authentication

## Database Configuration
- **Database Name**: `united_rehabs` (set in `/app/backend/.env` as `DB_NAME`)
- **Important**: Always ensure `DB_NAME=united_rehabs` in the backend .env file
